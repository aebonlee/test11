# P3BA35: 정치인 상세 페이지 하드코딩된 점수 제거

## 작업 개요
정치인 상세 페이지(`/politicians/[id]`)에서 하드코딩된 AI 점수, 카테고리 점수, 차트 데이터를 실제 API 데이터로 교체

## 작업 배경 및 필요성

### 문제점 분석 (2025-11-27)
프로덕션 사이트의 정치인 상세 페이지에서 가짜 하드코딩된 데이터가 표시됨

1. **AI_SCORES 하드코딩**
   - Claude: 970점, ChatGPT: 950점, Grok: 960점 (가짜 데이터)
   - V24.0 시스템에서는 Claude AI만 사용

2. **CATEGORY_SCORES 하드코딩**
   - 10개 카테고리 점수가 가짜 데이터로 표시
   - 실제 API에서 제공하는 `categoryScores` 사용 안함

3. **CHART_DATA_FULL 하드코딩**
   - 12개월 시계열 데이터가 가짜로 표시
   - API에서 시계열 데이터 미제공

## 해결 방안

### 1. AI_SCORES 하드코딩 제거
- 5개 AI 점수 배열 삭제
- V24.0 시스템은 Claude AI만 사용하므로 `politician.totalScore` 사용

### 2. CATEGORY_SCORES 교체
- 하드코딩된 점수 배열 → `CATEGORY_NAMES` 룩업 테이블로 변경
- API에서 받는 `politician.categoryScores` 사용

### 3. 상세평가보고서 구매 UI 단순화
- 5개 AI 선택 → Claude 1개만 선택 가능
- `handleToggleAll`, `totalPrice` 함수 단순화

### 4. 시계열 차트 처리
- API에서 시계열 데이터 미제공
- "점수 추이 차트 준비 중" 안내 메시지로 대체
