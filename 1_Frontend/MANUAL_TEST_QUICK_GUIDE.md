## 📱 테스트 시나리오

### 시나리오 1: 필터 태그 (5분)

**페이지**: `/politicians`

1. **필터 선택**
   ```
   [신분] → "현직" 선택
   ✅ "신분: 현직" 태그 나타남

   [출마직종] → "국회의원" 선택
   ✅ "출마직종: 국회의원" 태그 나타남

   [정당] → "더불어민주당" 선택
   ✅ "정당: 더불어민주당" 태그 나타남
   ```

2. **개별 제거**
   ```
   "신분: 현직" 태그의 X 버튼 클릭
   ✅ 신분 태그만 사라짐
   ✅ 다른 태그는 유지됨
   ```

3. **전체 초기화**
   ```
   "전체 초기화" 버튼 클릭
   ✅ 모든 태그 사라짐
   ✅ 필터가 리셋됨
   ```

4. **반응형 확인**
   ```
   뷰포트 320px로 변경
   ✅ 태그들이 자동으로 줄바꿈
   ✅ 레이아웃 깨지지 않음
   ```

**예상 소요 시간**: 3분

### 시나리오 3: FAB 버튼 (5분)

**페이지**: `/community`

1. **전체 탭 (모달 표시)**
   ```
   1. 커뮤니티 페이지 접속
   2. "전체" 탭 선택 (기본값)
   3. 우측 하단 + 버튼 클릭

   ✅ "카테고리 선택" 모달 표시
   ✅ "🏛️ 정치인 게시판" 옵션 보임
   ✅ "💬 회원 자유게시판" 옵션 보임
   ```

2. **정치인 게시판 탭**
   ```
   1. "🏛️ 정치인 게시판" 탭 클릭
   2. 우측 하단 + 버튼 클릭

   ✅ /community/posts/create-politician로 이동
   ✅ 모달 표시 안 됨 (직접 이동)
   ```

3. **회원 자유게시판 탭**
   ```
   1. "💬 회원 자유게시판" 탭 클릭
   2. 우측 하단 + 버튼 클릭

   ✅ /community/posts/create로 이동
   ✅ 모달 표시 안 됨 (직접 이동)
   ```

4. **FAB 고정 확인**
   ```
   페이지 스크롤 (위/아래)
   ✅ + 버튼 위치 고정 유지
   ✅ 항상 우측 하단에 표시
   ```

5. **터치 타겟 크기**
   ```
   개발자 도구에서 FAB 요소 검사
   ✅ width: 56px (w-14)
   ✅ height: 56px (h-14)
   ✅ 44px 이상 (WCAG 준수)
   ```

**예상 소요 시간**: 4분

## 🎯 터치 타겟 크기 검증 (5분)

### DevTools로 확인하는 방법

```javascript
// 1. 브라우저 콘솔에서 실행
document.querySelectorAll('button, a, input[type="checkbox"], input[type="radio"]').forEach(el => {
  const rect = el.getBoundingClientRect();
  if (rect.width < 44 || rect.height < 44) {
    console.warn('Too small:', el, `${rect.width}x${rect.height}`);
    el.style.outline = '2px solid red';
  } else {
    el.style.outline = '2px solid green';
  }
});
```

**예상 결과**:
- ✅ 모든 버튼이 녹색 테두리 (44px 이상)
- ❌ 빨간 테두리가 있다면 문제 발견
