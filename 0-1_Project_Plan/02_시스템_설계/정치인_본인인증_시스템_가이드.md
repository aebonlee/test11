# 정치인 이메일 인증 시스템 가이드

## 개요
PoliticianFinder 플랫폼에서 정치인은 **이메일 인증**을 통해 본인을 확인합니다.

---

## 핵심 원칙

### 인증 방식: 오직 이메일 인증만 사용
- 정치인이 등록한 이메일로 인증 링크 발송
- 인증 링크 클릭 시 본인 확인 완료
- **다른 방식 (이름/정당/직종 입력 등)은 사용하지 않음**

---

## 인증이 필요한 기능 (3가지)

### 1. 정치인 게시글 작성
- 정치인이 게시글을 작성할 때
- 이메일 인증 → 글쓰기 권한 부여

### 2. 정치인 댓글 작성
- 정치인이 댓글을 작성할 때
- 이메일 인증 → 댓글 작성 권한 부여

### 3. 상세평가 리포트 구매
- 정치인 본인만 자신의 보고서 구매 가능
- 이메일 인증 → 구매 권한 부여

---

## 인증 프로세스

### 1단계: 이메일 입력
정치인이 등록된 이메일 주소를 입력합니다.

### 2단계: 인증 링크 발송
시스템에서 해당 이메일로 인증 링크를 발송합니다.

### 3단계: 이메일 확인
정치인이 이메일에서 인증 링크를 클릭합니다.

### 4단계: 인증 완료
인증 완료 후 해당 기능 사용 권한이 부여됩니다.

---

## 기술 구현

### 백엔드 요구사항
1. 정치인 이메일 DB 테이블:
   - `politician_sessions` 테이블 사용
   - `email` 필드로 정치인 식별

2. 인증 요청 API:
   ```
   POST /api/politicians/verification/send-email
   {
     "email": "politician@example.com"
   }
   ```

3. 인증 확인 API:
   ```
   GET /api/politicians/verification/verify?token=xxx
   ```
