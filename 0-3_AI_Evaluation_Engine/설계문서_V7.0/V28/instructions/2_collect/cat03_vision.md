### 🚨🚨 콘텐츠 기반 과거 사건 필터링 (V28.1 필수!) 🚨🚨

**수집 전 내용(content)에서 과거 연도를 반드시 확인!**

```
⛔ 다음 패턴이 내용에 포함되면 절대 수집 금지:

1. 과거 연도 직접 언급 (OFFICIAL: 2022년 미만, PUBLIC: 2024년 미만):
   - "2002년", "2010년", "2015년" 등 기준 초과 연도 → ❌ 금지!
   - "10년 전", "20년 전", "수년 전" → ❌ 금지!
   - "과거", "예전", "당시" + 구체적 사건 → ❌ 금지!

2. 청문회/재조명에서 과거 사건 언급:
   - "청문회에서 ○○년 사건 질의" → ❌ 금지!
   - "과거 ○○ 의혹 재조명" → ❌ 금지!
   - "예전 ○○ 논란 재점화" → ❌ 금지!

3. 과거 판결/확정:
   - "○○년 유죄 확정" → ❌ 금지! (사건 연도 기준)
   - "과거 ○○ 혐의로 처벌" → ❌ 금지!

✅ 수집 가능 - 현재 시점 사건만:
   - "2024년 새로운 의혹 제기" → ✅ OK
   - "2025년 청문회 답변 태도 논란" → ✅ OK
   - "최근 정책 성과 발표" → ✅ OK
```

**핵심: data_content 내용을 읽고, 사건 발생 연도가 기준 외이면 수집하지 마세요!**

### 🚨🚨 과거 사건 언어 패턴 필터링 (V28.2 강화!) 🚨🚨

**연도 없이도 과거 사건임을 알 수 있는 표현 차단!**

```
⛔ 다음 표현이 포함되면 수집 금지 (과거 사건 신호):

1. 진행형/지속 표현 (= 오래된 사건의 증거):
   - "논란이 지속" / "의혹이 지속" / "논란이 계속"
   - "의혹이 계속 제기" / "비판이 이어지고"
   - "여전히 논란" / "아직도 해결되지 않은"
   - "지속적으로 제기되는" / "끊이지 않는 논란"

2. 추가/새로운 표현 (= 기존 사건이 오래됨):
   - "추가 의혹" / "추가 논란" / "추가 정황"
   - "새로운 정황이 드러나" / "새로운 증거"
   - "기존 논란을 넘어서" / "기존에 알려진"
   - "또다시 논란" / "다시 도마 위에"

3. 재조명/환기 표현:
   - "재조명" / "재점화" / "다시 주목"
   - "다시 거론" / "다시 언급" / "재부각"
   - "과거 행적" / "과거 발언" / "과거 이력"

💡 판단 원칙:
"이 표현이 나오면 사건이 처음 발생한 것이 아니다"
→ 처음이 아니면 = 과거 사건 = 기간 검증 대상!
```

### 🚨🚨 중복 수집 강화 규칙 (V28.2 필수!) 🚨🚨

**동일 사건의 다양한 표현을 1건으로 통합!**

```
⛔ 중복 수집 금지 - 핵심 규칙:

1. 동일 주체 + 동일 유형 = 1건만:
   ❌ "A 의혹" + "A 논란" + "A 의혹 지속"
   ✅ 위 3건은 모두 같은 사건 → 1건만 수집!

2. 핵심 키워드 중복 체크:
   - 비위 키워드: 표절, 비리, 횡령, 배임, 뇌물, 정치자금
   - 주체: 본인, 아들, 딸, 배우자, 가족
   - 같은 주체 + 같은 키워드 = 동일 사건!

3. 제목 변형 인식:
   ❌ "A 의혹" = "A 논란" = "A 의혹 제기" = "A 논란 지속"
   ✅ 모두 같은 사건 → 1건만!

4. 후속 보도 = 중복:
   - "○○ 의혹 추가 정황" → 원래 의혹과 동일
   - "○○ 논란 확산" → 원래 논란과 동일
   - "○○ 의혹 재점화" → 원래 의혹과 동일
```

## 8. 주의사항 (Pitfalls)

1. **공약과 비전 구분**: 단순 공약 ≠ 장기 비전
2. **실현 가능성 고려**: 허황된 약속 vs. 실현 가능한 비전
3. **시대 맥락**: 당시 시대 상황에서의 혁신성 평가
4. **구체성 확인**: 추상적 구호 vs. 구체적 계획

## 8-1. 중복 수집 금지 (CRITICAL!)

### 핵심 규칙
**동일한 사건/주제는 반드시 1개만 수집!**

### 금지 사항
- 같은 정책을 다른 날짜로 반복 수집 ❌
- 같은 비전을 다른 언론사로 반복 수집 ❌
- 같은 주제를 다른 표현으로 반복 수집 ❌

### 예시: 중복 수집 (잘못된 예)
```
[X] "2030 서울 비전 발표" (출처 A)
[X] "서울시 2030 발전 계획" (출처 B)
[X] "서울 미래 비전 2030" (출처 C)
→ 모두 같은 비전! 1개만 수집해야 함
```

### 동일 사건 판단 기준
- 같은 정책/비전에 대한 여러 보도
- 같은 계획의 다른 표현
- 후속 보도, 재조명 = 원래 사건과 동일

**중복 수집 시 전체 응답 무효 처리됨!**

```
[STEP 1] 웹검색 실행
         └→ "[정치인명] 비전" / "[정치인명] 정책" 등 검색
         └→ 검색 결과에서 실제 기사 URL 확보

[STEP 2] 기사 확인
         └→ 검색된 URL이 실제 존재하는지 확인
         └→ 기사 내용 읽고 정치인과 관련 있는지 확인

[STEP 3] 데이터 추출
         └→ 확인된 기사에서 제목/내용/날짜 추출
         └→ 실제 기사 URL을 source_url에 기록

[STEP 4] JSON 생성
         └→ 검색으로 찾은 실제 데이터만 포함
```

### 웹검색 키워드 가이드 (비전)

```
긍정 검색어:
- "[정치인명] 비전 제시"
- "[정치인명] 미래 정책"
- "[정치인명] 혁신 정책"
- "[정치인명] 장기 계획"

부정 검색어:
- "[정치인명] 비전 부재"
- "[정치인명] 정책 실패"
- "[정치인명] 포퓰리즘"
- "[정치인명] 비현실적"

중립 검색어:
- "[정치인명] 공약"
- "[정치인명] 정책 발표"
- "[정치인명] 국정 비전"
```

### 🚨 핵심 규칙

```
1. 웹검색 없이 수집한 데이터 = 무효!
2. 검색 결과에 없는 정보 = 수집 금지!
3. URL 접속 불가능 = 해당 데이터 삭제!
4. AI 기억으로 만든 스캔들 = 날조 = 절대 금지!
```

**웹검색 규칙 위반 시 전체 수집 데이터 무효 처리!**
