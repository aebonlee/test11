### 🚨🚨 과거 사건 언어 패턴 필터링 (V28.2 강화!) 🚨🚨

**연도 없이도 과거 사건임을 알 수 있는 표현 차단!**

```
⛔ 다음 표현이 포함되면 수집 금지 (과거 사건 신호):

1. 진행형/지속 표현 (= 오래된 사건의 증거):
   - "논란이 지속" / "의혹이 지속" / "논란이 계속"
   - "의혹이 계속 제기" / "비판이 이어지고"
   - "여전히 논란" / "아직도 해결되지 않은"
   - "지속적으로 제기되는" / "끊이지 않는 논란"

2. 추가/새로운 표현 (= 기존 사건이 오래됨):
   - "추가 의혹" / "추가 논란" / "추가 정황"
   - "새로운 정황이 드러나" / "새로운 증거"
   - "기존 논란을 넘어서" / "기존에 알려진"
   - "또다시 논란" / "다시 도마 위에"

3. 재조명/환기 표현:
   - "재조명" / "재점화" / "다시 주목"
   - "다시 거론" / "다시 언급" / "재부각"
   - "과거 행적" / "과거 발언" / "과거 이력"

💡 판단 원칙:
"이 표현이 나오면 사건이 처음 발생한 것이 아니다"
→ 처음이 아니면 = 과거 사건 = 기간 검증 대상!
```

### 🚨🚨 중복 수집 강화 규칙 (V28.2 필수!) 🚨🚨

**동일 사건의 다양한 표현을 1건으로 통합!**

```
⛔ 중복 수집 금지 - 핵심 규칙:

1. 동일 주체 + 동일 유형 = 1건만:
   ❌ "A 의혹" + "A 논란" + "A 의혹 지속"
   ✅ 위 3건은 모두 같은 사건 → 1건만 수집!

2. 핵심 키워드 중복 체크:
   - 비위 키워드: 표절, 비리, 횡령, 배임, 뇌물, 정치자금
   - 주체: 본인, 아들, 딸, 배우자, 가족
   - 같은 주체 + 같은 키워드 = 동일 사건!

3. 제목 변형 인식:
   ❌ "A 의혹" = "A 논란" = "A 의혹 제기" = "A 논란 지속"
   ✅ 모두 같은 사건 → 1건만!

4. 후속 보도 = 중복:
   - "○○ 의혹 추가 정황" → 원래 의혹과 동일
   - "○○ 논란 확산" → 원래 논란과 동일
   - "○○ 의혹 재점화" → 원래 의혹과 동일
```

## 8. 주의사항 (Pitfalls)

1. **학벌 과대평가 금지**: 명문대 출신이라도 경력이 짧으면 객관적으로 기록
2. **경력 허수 구별**: 명목상 경력 vs. 실질적 경력 구분
3. **무관 자격증 제외**: 정치와 무관한 자격증은 수집하지 않음
4. **시대 고려**: 과거 교육 환경 고려 (예: 고졸도 당시엔 평균 이상)
5. **학력 위조 확인**: 의혹과 확정 판결 구분하여 기록

## 8-1. 중복 수집 금지 (CRITICAL!)

### 핵심 규칙
**동일한 사건/주제는 반드시 1개만 수집!**

### 금지 사항
- 같은 사건을 다른 날짜로 반복 수집 ❌
- 같은 사건을 다른 언론사로 반복 수집 ❌
- 같은 주제를 다른 표현으로 반복 수집 ❌

### 예시: 중복 수집 (잘못된 예)
```
[X] "서울대 법학과 졸업" (출처 A)
[X] "서울대학교 법학과 학사" (출처 B)
[X] "서울대 법대 졸업" (출처 C)
→ 모두 같은 학력! 1개만 수집해야 함
```

### 예시: 올바른 수집
```
[O] "서울대 법학과 졸업 (1985)" - 1개만!
[O] "변호사 자격증 취득 (1988)" - 별개 항목이므로 OK
[O] "행정고시 합격 (1987)" - 별개 항목이므로 OK
```

### 동일 사건 판단 기준
- 같은 학력/자격증/경력 정보
- 같은 법안 발의 건
- 후속 보도, 재조명 = 원래 사건과 동일

### 체크 방법
수집 전 스스로 확인:
1. "이 주제를 이미 수집했나?"
2. "비슷한 제목의 항목이 있나?"
3. "같은 사건의 다른 보도인가?"

**중복 수집 시 전체 응답 무효 처리됨!**

## 8-3. 🚨🚨🚨 V28.6 웹검색 필수 규칙 (CRITICAL!) 🚨🚨🚨

### ⛔ AI 기억 기반 수집 절대 금지!

```
┌─────────────────────────────────────────────────────────────┐
│ 🚫 절대 금지: AI가 기억/학습 데이터로 내용 생성            │
│ 🚫 절대 금지: AI가 URL을 추측/생성                         │
│ 🚫 절대 금지: 존재 여부 확인 안 된 정보 수집               │
│                                                             │
│ ✅ 필수: 웹검색으로 실제 기사 먼저 찾기                    │
│ ✅ 필수: 검색 결과에서 URL 확보 후 수집                    │
│ ✅ 필수: 실제 존재하는 기사만 데이터로 변환                │
└─────────────────────────────────────────────────────────────┘
```

### 올바른 수집 프로세스

```
[STEP 1] 웹검색 실행
         └→ "[정치인명] 전문성" / "[정치인명] 학력" 등 검색
         └→ 검색 결과에서 실제 기사 URL 확보

[STEP 2] 기사 확인
         └→ 검색된 URL이 실제 존재하는지 확인
         └→ 기사 내용 읽고 정치인과 관련 있는지 확인

[STEP 3] 데이터 추출
         └→ 확인된 기사에서 제목/내용/날짜 추출
         └→ 실제 기사 URL을 source_url에 기록

[STEP 4] JSON 생성
         └→ 검색으로 찾은 실제 데이터만 포함
```

### 웹검색 키워드 가이드 (전문성)

긍정 검색어:

부정 검색어:

- "[정치인명] 무능"

중립 검색어:

## 9. JSON 출력 형식

```json
{
  "items": [
    {
      "item_num": 1,
      "data_title": "간결한 제목 (20자 이내)",
      "data_content": "구체적 내용 (사실 기반, 100-300자)",
      "data_source": "구체적 출처명",
      "source_url": "https://...",
      "source_type": "OFFICIAL",
      "data_date": "2025-06-15"
    }
  ]
}
```

### 필드 설명
| 필드 | 설명 | 필수 |
|------|------|------|
| item_num | 항목 번호 (1~50) | O |
| data_title | 간결한 제목 (20자 이내) | O |
| data_content | 구체적 내용 (100-300자) | O |
| data_source | 출처명 (기관/언론사명) | O |
| source_url | 실제 존재하는 URL | O |
| source_type | "OFFICIAL" 또는 "PUBLIC" | O |
| data_date | YYYY-MM-DD 형식 | O |

### 주의
- **rating 필드 포함하지 마세요!** 수집 단계에서는 평가 없음
- source_type은 반드시 대문자
