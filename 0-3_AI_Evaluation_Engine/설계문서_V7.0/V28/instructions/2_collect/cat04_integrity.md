### 🚨 청문회/재조명 기사 처리 규칙 (CRITICAL!)

**기사 게시일 ≠ 사건 발생일**

```
❌ 수집 금지 예시:
- 2025년 청문회에서 "2002년 정치자금 수수" 언급 → 사건은 2002년 → 수집 금지!
- 2025년 기사: "20년 전 비리 재조명" → 사건은 20년 전 → 수집 금지!
- 2024년 보도: "과거 재산 형성 의혹" (2010년 사건) → 수집 금지!

✅ 수집 가능 예시:
- 2025년 청문회에서 "자료 제출 거부" → 거부 행위가 2025년 → 수집 OK
- 2024년 새로운 비리 의혹 제기 → 사건이 2024년 → 수집 OK
- 청문회 "답변 태도" 논란 → 태도가 현재 시점 → 수집 OK

⚠️ 구분 기준:
- "과거 사건 자체"를 보도 → 사건 발생 시점 기준 → 기간 초과면 금지
- "과거 사건에 대한 현재 대응"을 보도 → 대응 시점 기준 → 수집 가능
```

### 🚨🚨 콘텐츠 기반 과거 사건 필터링 (V28.1 필수!) 🚨🚨

**수집 전 내용(content)에서 과거 연도를 반드시 확인!**

```
⛔ 다음 패턴이 내용에 포함되면 절대 수집 금지:

1. 과거 연도 직접 언급:
   - "2002년 정치자금 수수" → ❌ 금지! (2002년 사건)
   - "2010년 재산 형성 의혹" → ❌ 금지! (2010년 사건)
   - "10년 전 비리" → ❌ 금지!
   - "과거 뇌물 수수" → ❌ 금지!

2. "과거" 키워드 + 비위 사건:
   - "과거 정치자금 의혹이 지속" → ❌ 금지!
   - "예전 비리가 재조명" → ❌ 금지!
   - "당시 횡령 혐의" → ❌ 금지!

3. 오래된 판결/확정:
   - "2002년 유죄 확정" → ❌ 금지!
   - "과거 벌금형 선고" → ❌ 금지!

✅ 수집 가능 (현재 시점 사건):
   - "2024년 새로운 비리 의혹 제기" → ✅ OK
   - "2025년 청문회 답변 태도 논란" → ✅ OK
   - "최근 정치자금 내역 공개" → ✅ OK
```

**핵심: data_content 내용을 읽고, 사건 발생 연도가 기준 외이면 수집하지 마세요!**

### 🚨🚨 과거 사건 언어 패턴 필터링 (V28.2 강화!) 🚨🚨

**연도 없이도 과거 사건임을 알 수 있는 표현 차단!**

```
⛔ 다음 표현이 포함되면 수집 금지 (과거 사건 신호):

1. 진행형/지속 표현 (= 오래된 사건의 증거):
   - "논란이 지속" / "의혹이 지속" / "논란이 계속"
   - "의혹이 계속 제기" / "비판이 이어지고"
   - "여전히 논란" / "아직도 해결되지 않은"
   - "지속적으로 제기되는" / "끊이지 않는 논란"

2. 추가/새로운 표현 (= 기존 사건이 오래됨):
   - "추가 의혹" / "추가 논란" / "추가 정황"
   - "새로운 정황이 드러나" / "새로운 증거"
   - "기존 논란을 넘어서" / "기존에 알려진"
   - "또다시 논란" / "다시 도마 위에"

3. 재조명/환기 표현:
   - "재조명" / "재점화" / "다시 주목"
   - "다시 거론" / "다시 언급" / "재부각"
   - "과거 행적" / "과거 발언" / "과거 이력"

💡 판단 원칙:
"이 표현이 나오면 사건이 처음 발생한 것이 아니다"
→ 처음이 아니면 = 과거 사건 = 기간 검증 대상!
```

### 🚨🚨 중복 수집 강화 규칙 (V28.2 필수!) 🚨🚨

**동일 사건의 다양한 표현을 1건으로 통합!**

```
⛔ 중복 수집 금지 - 핵심 규칙:

1. 동일 주체 + 동일 유형 = 1건만:
   ❌ "A 의혹" + "A 논란" + "A 의혹 지속"
   ✅ 위 3건은 모두 같은 사건 → 1건만 수집!

2. 핵심 키워드 중복 체크:
   - 비위 키워드: 표절, 비리, 횡령, 배임, 뇌물, 정치자금
   - 주체: 본인, 아들, 딸, 배우자, 가족
   - 같은 주체 + 같은 키워드 = 동일 사건!

3. 제목 변형 인식:
   ❌ "A 의혹" = "A 논란" = "A 의혹 제기" = "A 논란 지속"
   ✅ 모두 같은 사건 → 1건만!

4. 후속 보도 = 중복:
   - "○○ 의혹 추가 정황" → 원래 의혹과 동일
   - "○○ 논란 확산" → 원래 논란과 동일
   - "○○ 의혹 재점화" → 원래 의혹과 동일
```

## 8. 주의사항 (Pitfalls)

1. **의혹 vs 확정 구분**: 수사 중 vs 유죄 확정 명확히 구분
2. **정치적 공격 구분**: 정쟁성 공격 vs 실제 비리 구별
3. **금액 규모 기록**: 비리 금액 규모 함께 기록
4. **시기 명시**: 사건 발생 시기 명확히 기록
5. **결과 추적**: 수사/재판 결과 함께 기록

## 8-1. 중복 수집 금지 (CRITICAL!)

### 핵심 규칙
**동일한 사건/주제는 반드시 1개만 수집!**

### 금지 사항
- 같은 사건을 다른 날짜로 반복 수집 ❌
- 같은 사건을 다른 언론사로 반복 수집 ❌
- 같은 주제를 다른 표현으로 반복 수집 ❌

### 예시: 중복 수집 (잘못된 예)
```
[X] "손길승 불법자금 수수 논란 재점화" (2024-09-10)
[X] "손길승 회장 불법 자금 수수" (2022-03-01)
[X] "손길승 불법 정치자금 사건" (2025-01-05)
→ 모두 같은 사건! 1개만 수집해야 함
```

### 예시: 올바른 수집
```
[O] "손길승 불법자금 수수 (2002년 유죄 확정)" - 1개만!
[O] "인사청문회 재산 형성 의혹" - 별개 사건이므로 OK
[O] "아들 표절 의혹" - 별개 사건이므로 OK
```

## 8-2. URL 규칙 (CRITICAL!)

### 핵심 규칙
**출처 유형에 따라 URL 필수 여부가 다름!**

### SNS (URL 불필요)
다음 플랫폼은 source_url 빈 값 허용:
- 트위터/X
- 페이스북
- 유튜브
- 인스타그램

### 웹사이트 (URL 필수!)
다음 출처는 **반드시 실제 URL 제공**:
- 언론사 기사 (조선, 중앙, 한겨레 등)
- 위키 (나무위키, 위키피디아)
- 시민단체 보고서
- 커뮤니티 게시물
- 기타 모든 웹 기반 출처

### 금지 사항
- ❌ URL을 모르면서 데이터 수집 금지
- ❌ 가짜 URL 생성 절대 금지
  - `/articles/123456` 같은 패턴
  - `example.com` 사용
  - 존재하지 않는 URL 조작

### 예시
```
[O] SNS: source_url = "" (빈 값 OK)
[O] 언론: source_url = "https://www.chosun.com/..." (실제 URL)
[X] 언론: source_url = "" (URL 없으면 수집 금지!)
[X] 가짜: source_url = "https://news.com/articles/123" (가짜 URL!)
```

**URL 규칙 위반 시 전체 응답 무효 처리됨!**

### 올바른 수집 프로세스

```
[STEP 1] 웹검색 실행
         └→ "김민석 정치자금" / "김민석 청렴" 등 검색
         └→ 검색 결과에서 실제 기사 URL 확보

[STEP 2] 기사 확인
         └→ 검색된 URL이 실제 존재하는지 확인
         └→ 기사 내용 읽고 정치인과 관련 있는지 확인

[STEP 3] 데이터 추출
         └→ 확인된 기사에서 제목/내용/날짜 추출
         └→ 실제 기사 URL을 source_url에 기록

[STEP 4] JSON 생성
         └→ 검색으로 찾은 실제 데이터만 포함
```
