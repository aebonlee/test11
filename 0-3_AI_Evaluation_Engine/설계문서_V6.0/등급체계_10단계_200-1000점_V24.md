## 점수 계산 방식

### 1단계: 카테고리별 점수 계산
```
Category_Score = (6.0 + avg_rating × 0.5) × 10

여기서:
- avg_rating: 데이터 평점 평균 (-8 ~ +8)
- Coefficient: 0.5
- Prior: 6.0 (선출직 정치인 기본 신뢰)

범위: [20, 100]
- 최소: (6.0 + (-8 × 0.5)) × 10 = 20점
- 최대: (6.0 + (+8 × 0.5)) × 10 = 100점
```

### 2단계: 최종 점수 계산
```
Final_Score = round(min(Σ(10개 카테고리 점수), 1000))

범위: [200, 1000]
- 최소: 20 × 10 = 200점
- 최대: 100 × 10 = 1,000점
```

## 등급 판정 알고리즘 (Python)

```python
def get_grade_v24(final_score):
    """
    최종 점수를 10단계 등급으로 변환 (200-1,000점)

    Args:
        final_score: 최종 점수 (200~1,000점)

    Returns:
        tuple: (등급 심볼, 등급 이름, 이모지)
    """
    if final_score >= 920:
        return ('M', 'Mugunghwa', '🌺')
    elif final_score >= 840:
        return ('D', 'Diamond', '💎')
    elif final_score >= 760:
        return ('E', 'Emerald', '💚')
    elif final_score >= 680:
        return ('P', 'Platinum', '💠')
    elif final_score >= 600:
        return ('G', 'Gold', '🏅')
    elif final_score >= 520:
        return ('S', 'Silver', '🥈')
    elif final_score >= 440:
        return ('B', 'Bronze', '🥉')
    elif final_score >= 360:
        return ('I', 'Iron', '🔩')
    elif final_score >= 280:
        return ('Tn', 'Tin', '⚙️')
    else:
        return ('L', 'Lead', '📉')
```

## 등급별 특성

### 상위 등급 (M, D, E)
- **M (920-1,000점)**: 모든 분야에서 최우수 수준
- **D (840-919점)**: 대부분 분야에서 우수 수준
- **E (760-839점)**: 전반적으로 양호한 수준

### 중위 등급 (P, G, S)
- **P (680-759점)**: 보통 이상의 수준
- **G (600-679점)**: 보통 수준
- **S (520-599점)**: 보통 이하 수준

### 하위 등급 (B, I, Tn, L)
- **B (440-519점)**: 개선이 필요한 수준
- **I (360-439점)**: 상당한 개선이 필요한 수준
- **Tn (280-359점)**: 근본적인 개선이 필요한 수준
- **L (200-279점)**: 전면적인 개선이 필요한 수준
