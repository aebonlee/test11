## V13.0 알고리즘 적용

### 점수 계산 공식
```python
PRIOR = 6.5
COEFFICIENT = 0.5

category_score = (6.5 + avg_rating × 0.5) × 10
final_score = sum(10개 카테고리 점수)
```

### 점수 범위
```
최소 점수 (모든 rating = -7):
  카테고리 점수 = (6.5 + (-7) × 0.5) × 10 = 30점
  최종 점수 = 30 × 10 = 300점 (이론적 최저)

중간 점수 (모든 rating = 0):
  카테고리 점수 = (6.5 + 0 × 0.5) × 10 = 65점
  최종 점수 = 65 × 10 = 650점

최대 점수 (모든 rating = +7):
  카테고리 점수 = (6.5 + 7 × 0.5) × 10 = 100점
  최종 점수 = 100 × 10 = 1,000점
```

**실제 운영 범위**: 400~1,000점 (등급 체계)

## 점수 계산 흐름 (V13.0)

### 1단계: 데이터 수집 + Rating
```
Claude Haiku API가 데이터 수집 → Rating 부여 (-7 ~ +7)
카테고리당 50개 데이터 수집
```

### 2단계: 카테고리별 평균 Rating 계산
```python
# 각 카테고리별 50개 rating의 평균
avg_rating = sum(50개 ratings) / 50
```

### 3단계: 카테고리 점수 계산 (V13.0 알고리즘)
```python
category_score = (6.5 + avg_rating × 0.5) × 10
# 범위: 30~100점
```

### 4단계: 최종 점수 계산
```python
final_score = sum(10개 카테고리 점수)
# 범위: 300~1,000점 (등급 체계: 400~1,000점)
```

### 5단계: 등급 부여
```python
if 900 <= score <= 1000:
    grade = 'I'  # Iridium 💠
elif 800 <= score < 900:
    grade = 'B'  # Bronze 🥉
elif 700 <= score < 800:
    grade = 'S'  # Silver 🥈
elif 600 <= score < 700:
    grade = 'G'  # Gold 🥇
elif 500 <= score < 600:
    grade = 'P'  # Platinum 🏆
elif 450 <= score < 500:
    grade = 'E'  # Emerald 💚
elif 425 <= score < 450:
    grade = 'D'  # Diamond 💎
else:  # 400 <= score < 425
    grade = 'M'  # Mithril 🌺
```

## V6.2 → V13.0 변경사항

| 항목 | V6.2 | V13.0 |
|------|------|------|
| Prior | 7.0 | 6.5 ✅ |
| Coefficient | 0.6 | 0.5 ✅ |
| Rating 범위 | -5 ~ +5 (11단계) | -7 ~ +7 (15단계) ✅ |
| 모델 | Sonnet 4.5 | Haiku 3.5 ✅ |
| 카테고리당 데이터 | 70~100개 | 50개 (고정) ✅ |
| 재수집 로직 | 없음 | 3차 시도 ✅ |
| 중복 방지 | 없음 | 다른 키워드/출처 ✅ |

### 등급 순서 변경 (V6.2 → V13.0)
```
V6.2: M(최고) → D → E → P → G → S → B → I(최저)
V13.0: I(최고) → B → S → G → P → E → D → M(최저)
```

**변경 이유**: 알파벳 역순이 직관적이지 않아 I(Iridium, 최고가 금속)를 최상위로 변경
