## V15.0 알고리즘 적용

### 점수 계산 공식
```python
PRIOR = 6.0
COEFFICIENT = 0.5
RATING_RANGE = -6 ~ +10 (17단계)
MAX_FINAL_SCORE = 1000

category_score = (6.0 + avg_rating × 0.5) × 10
raw_final_score = sum(10개 카테고리 점수)
final_score = min(raw_final_score, 1000)  # 1,000점 상한 적용
```

### 점수 범위
```
최소 점수 (모든 rating = -6):
  카테고리 점수 = (6.0 + (-6) × 0.5) × 10 = 30점
  최종 점수 = 30 × 10 = 300점

중간 점수 (모든 rating = +2):
  카테고리 점수 = (6.0 + 2 × 0.5) × 10 = 70점
  최종 점수 = 70 × 10 = 700점

최대 점수 (모든 rating = +10):
  카테고리 점수 = (6.0 + 10 × 0.5) × 10 = 110점
  이론적 최종 점수 = 110 × 10 = 1,100점
  → 실제 최종 점수 = min(1,100, 1,000) = 1,000점 ✅
```

**실제 운영 범위**: 300~1,000점 (등급 체계: 400~1,000점)

## V15.0 주요 변경사항

### V14.0 → V15.0 비교

| 항목 | V14.0 | V15.0 | 변경 이유 |
|------|------|------|-----------|
| **Prior** | 6.0 | 6.0 | 유지 |
| **Coefficient** | 0.5 | 0.5 | 유지 |
| **Rating 범위** | -8 ~ +8 | **-6 ~ +10** | 긍정 범위 확대 |
| **Rating 단계** | 17단계 (대칭) | 17단계 (**비대칭**) | 우수 사례 세밀 구분 |
| **이론적 최대** | 1,000점 | 1,100점 | Rating +10 영향 |
| **실제 최대** | 1,000점 | **1,000점 (상한 적용)** | 등급 체계 일관성 |
| **최소 점수** | 220점 | 300점 | Rating 하한 -6으로 상향 |

### 핵심 개선사항

1. **긍정 범위 확대**: -8~+8 → -6~+10
   - 우수 사례를 더 세밀하게 구분 (+7, +8, +9, +10)
   - 극단적 부정 완화 (-8, -7 제거)

2. **1,000점 상한 적용**:
   - 이론적으로 1,100점 가능하지만 실제로는 1,000점으로 제한
   - 등급 체계 일관성 유지
   - 역대 최고 정치인도 1,000점 이내로 평가

3. **등급 분포 자연스러움**:
   - Prior 6.0 유지로 중앙값 안정
   - Rating 0점 = 카테고리 60점 = 최종 600점 (G등급)

## 점수 계산 흐름 (V15.0)

### 1단계: 데이터 수집 + Rating
```
Claude Haiku API가 데이터 수집 → Rating 부여 (-6 ~ +10)
카테고리당 50개 데이터 수집
재수집: 50개 미달 시 최대 3차 시도
```

### 2단계: 카테고리별 평균 Rating 계산
```python
# 각 카테고리별 50개 rating의 평균
avg_rating = sum(50개 ratings) / 50
```

### 3단계: 카테고리 점수 계산 (V15.0 알고리즘)
```python
category_score = (6.0 + avg_rating × 0.5) × 10
# 범위: 30~110점 (이론적)
# 실제: 30~100점 (각 카테고리는 100점 상한 없음)
```

### 4단계: 최종 점수 계산
```python
raw_final_score = sum(10개 카테고리 점수)
# 범위: 300~1,100점 (이론적)

# V15.0 핵심: 1,000점 상한 적용
final_score = min(raw_final_score, 1000)
# 실제: 300~1,000점
```

### 5단계: 등급 부여
```python
if 900 <= score <= 1000:
    grade = 'I'  # Iridium 💠
elif 800 <= score < 900:
    grade = 'B'  # Bronze 🥉
elif 700 <= score < 800:
    grade = 'S'  # Silver 🥈
elif 600 <= score < 700:
    grade = 'G'  # Gold 🥇
elif 500 <= score < 600:
    grade = 'P'  # Platinum 🏆
elif 450 <= score < 500:
    grade = 'E'  # Emerald 💚
elif 425 <= score < 450:
    grade = 'D'  # Diamond 💎
else:  # 300 <= score < 425
    grade = 'M'  # Mithril 🌺
```

## 1,000점 상한 적용 이유

### 왜 1,100점을 허용하지 않는가?

1. **등급 체계 일관성**:
   - V13.0, V14.0 모두 1,000점 기준
   - 등급 체계 변경 시 혼란 초래

2. **현실적 도달 가능성**:
   - 모든 카테고리에서 평균 +10 받기는 사실상 불가능
   - "완벽한 정치인"은 이론적 개념일 뿐

3. **등급 상징성**:
   - 1,000점 = 최고 등급 I의 상징적 의미
   - 1,100점 등장 시 등급 체계 재조정 필요

4. **점수 압축 효과**:
   - 1,000점 이상은 모두 "역대 최고 수준"
   - 세밀한 구분이 무의미함
